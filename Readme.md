# 📄 Web 简历系统（Resume Web Project）

## 一、项目概述

本项目为一个基于 Web 的个人简历系统，按照**前端技术能力的逐步提升路径**进行阶段性设计与实现。

项目从最基础的 HTML 与 CSS 布局入手，逐步引入现代布局方案、交互逻辑以及前端框架，用于系统性训练前端开发中的**结构设计、布局理解与工程化演进能力**。

该项目定位为：

- 一个可持续迭代的前端练习项目
- 一个能够真实反映前端基础与成长过程的工程示例

---

## 二、设计目标与思路

在实际学习与项目实践中，发现很多前端练习项目存在以下问题：

- 一次性完成，缺乏版本演进
- 技术堆叠但缺乏设计动机
- 难以体现对基础原理的理解程度

因此，本项目在设计之初即明确采用**分阶段实现、逐步演进**的方式，每一阶段只引入当前阶段所需的技术能力，确保每一步都有明确的学习目标与技术侧重点。

---

## 三、技术演进与版本规划

### v1.0（当前版本）

**HTML + CSS（基于 float 的传统布局方案）**

设计说明：

- 使用语义化 HTML 构建页面结构，明确各模块职责
- 使用 CSS float 实现经典两栏布局（左侧信息栏 + 右侧内容栏）
- 通过处理浮动高度塌陷等问题，加深对传统布局机制的理解
- 使用标准 HTML form 完成简历信息填写与提交流程
- 页面结构在设计时即考虑后续布局重构与功能扩展的可能性

**设计目的**：
通过传统布局方式，扎实理解页面结构与 CSS 布局原理，而非直接依赖现代布局方案。

## 四、当前版本实现情况（v1.0）

### 技术栈

- HTML5
- CSS（float 布局）
- 原生 HTML 表单机制

### 已实现内容

- 简历信息填写模块（个人信息、教育经历、项目经历、技术栈）
- 一寸照片上传并随表单提交

- 模块化页面结构设计，便于后续重构

### 当前版本说明

当前版本重点放在**页面结构与布局原理的理解与实现**上，暂未引入复杂交互逻辑。
后续版本将逐步通过 JavaScript 与框架技术对该问题进行完善。

---

## 五、模块划分与结构设计

项目当前包含以下核心模块：

- **个人信息模块**：姓名、年龄、联系方式、邮箱、户籍等基础信息
- **照片模块**：用于照片展示与上传，为后续预览功能预留接口
- **技术栈模块**：展示掌握的前端相关技术与工具
- **项目经历模块**：展示项目背景、职责与使用技术
- **教育经历模块**：展示教育背景与相关说明

模块之间保持结构独立，避免强耦合，便于后续进行布局与框架层面的升级。
