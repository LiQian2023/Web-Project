# 🚀 排序算法可视化系统（Sorting Visualization System）

> 一个基于原生 JavaScript 实现的排序算法可视化与性能分析系统
> 支持多算法同步对比、状态驱动动画、三维性能分析

---

## 📌 项目简介

本项目通过可视化方式展示经典排序算法的执行过程，并结合性能统计模块，实现算法执行过程的**可观测化与可对比分析**。

不仅关注“结果”，更关注：

- 排序过程如何发生
- 每一步操作如何执行
- 不同算法之间的性能差异

---

## 🎯 核心功能

### 1️⃣ 排序算法可视化

支持算法：

- 冒泡排序（Bubble Sort）
- 插入排序（Insertion Sort）
- 选择排序（Selection Sort）

可视化特性：

- 动态柱状图展示数组状态
- 高亮当前比较元素（active）
- 已排序元素标记（sorted）
- 临时元素状态（tmp）

---

### 2️⃣ 动画执行系统（核心亮点）

将排序过程拆解为原子操作：

- `compare`：比较
- `swap`：交换
- `move`：移动
- `insert`：插入

通过步骤驱动模型：

```
排序算法 → steps（指令流）→ 调度执行 → UI渲染
```

实现：

- 算法逻辑与 UI 解耦
- 可控动画流程
- 可扩展动画机制

---

### 3️⃣ 多算法同步对比

- 三种排序算法在不同视口中**同步执行**
- 使用 `Promise.all` 实现并发调度
- 保证动画节奏一致

---

### 4️⃣ 性能分析模块

构建三维性能分析体系：

| 维度     | 说明                |
| -------- | ------------------- |
| 算法时间 | 纯算法执行耗时      |
| 动画时间 | 可视化执行耗时      |
| 操作统计 | 比较次数 / 交换次数 |

实现方式：

- `performance.now()` 精确计时
- 独立算法模块（无 UI 干扰）

---

### 5️⃣ 状态驱动渲染模型

核心状态：

- `active`：当前操作元素
- `sorted`：已排序区域
- `tmp`：临时缓冲区（插入排序）

实现：

```
状态变化 → 渲染更新 → 动画执行
```

---

## 🧠 核心设计思想

### ✅ 1. 指令流（Steps）驱动

- 排序算法只负责生成步骤
- 渲染系统负责执行步骤

👉 解耦算法与 UI

---

### ✅ 2. 三层架构

```
算法层 → 步骤层 → 渲染层
```

优势：

- 可维护性强
- 易扩展新算法
- 动画逻辑统一管理

---

### ✅ 3. FLIP 动画思想

通过位置差计算实现平滑动画：

```
First → Last → Invert → Play
```

---

### ✅ 4. 异步流程控制

- 使用 `async / await`
- 支持暂停 / 继续
- 支持多视口同步执行

---

## 排序动画优化

### 冒泡排序

冒泡排序两步动画：比较与交换

- pick:
    - 抬起被选中的两根柱子
- compare:
    - 高亮当前比较的两根柱子
- swap：
    - 高柱子右移，矮柱子左移
- down：
    - 落下完成交换/比较的两根柱子

### 插入排序

插入排序四步动画：选择关键字 比较 移动 插入

- pick：
    - 抬起被选中的柱子
- move:
    - 将抬起的柱子右移到待插入区域
- pick：
    - 抬起被选中的柱子
- compare：
    - 高亮当前比较的两根柱子
- move：
    - 右移大于关键字的柱子
- move：
    - 左移关键字柱子到当前空位
- down:
    - 下落关键字柱子

### 选择排序

选择排序三步动画：选择最小值 比较 交换

- pick：
    - 抬起被选中的柱子
- compare：
    - 高亮当前比较的两根柱子
- select：
    - 特殊高亮当前更小的柱子并
- swap：
    - 交换特殊高亮柱子与起点柱子

## 重构排序算法

### 旧版本问题

- 排序算法与动画逻辑高度耦合
- 算法执行过程中直接操作 DOM，导致：
    - 状态不可控
    - 调试困难
    - 扩展性差

---

### 新版本架构

#### 1. 指令流设计（核心）

- 排序算法仅负责记录排序过程
- 将排序行为抽象为步骤序列（steps）
- 每个 step 描述一个最小操作（如 compare / swap）

✔ 实现完成

---

#### 2. 执行调度器（runSteps）

- 负责按顺序执行 steps
- 核心职责：
    - 遍历步骤
    - 清除上一帧状态
    - 应用当前步骤状态
    - 控制执行节奏（delay）

✔ 基础实现完成

---

#### 3. 动画控制

- 当前通过 class 控制动画效果
- 动画执行逻辑尚未完全抽离为独立 animate 模块

##### 交换逻辑

通过下标完成交换：indices[a, b]
5, 4, 3, 2, 1
5px 288px 5px 288px 5px 288px 5px 288px 5px 288px 5px

- 柱子宽度：barWidth
- 柱子间隔：gap
- a的起点：x1 = （gap + barWidth）\* i + gap -> 5px, 0
- b的起点：x2 = （gap + barWidth）\* j + gap -> 298px, 0
- a右移：translateX(x2 - x1);
- b左移：translateX(x1 - x2);

---

#### 4. 暂停 / 继续机制

- 当前支持暂停与继续
- 实现方式：
    - 在 delay 前检查 isPaused

- 存在问题：
    - 暂停为“延迟生效”（非即时暂停）

⚠️ 待优化（需实现可中断 delay）
