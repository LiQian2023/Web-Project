需求文档

页面布局

页面分为四部分：
  1. 页面头部
  2. 页面导航
  3. 页面主体
  4. 页面底部
整个网页使用的响应式布局，通过媒体请求 @media 规则来实现响应式页面

核心模块

在不同的页面部分，所具备的模块也是不相同的：
页面头部
logo模块：这里我们实现采用 Noah 的 Web 前端之旅
查询模块：通过搜索框，实现搜索功能
菜单模块：通过按钮实现下拉菜单
页面导航
横向导航栏
页面主体
左侧导航：左侧导航栏为正常的垂直导航栏
内容主体：这里就是正常的页面内容包括3个部分：
内容介绍部分
分页栏
笔记分享与展示
右侧下拉导航：右侧导航栏为隐藏式下拉导航，或者说是提示工具导航
页面底部
正常的页面备注，如备案号等，这里我们在该部分展示个人的博客、git仓库等信息

实现步骤

采取先整体后局部的设计思路：
  1. 实现页面的整体布局
  2. 实现每一部分的模块内容
  3. 进行每一部分的样式调整
  4. 实现响应式

响应式页面需求

响应式页面，在具体设计的过程中，可以通过不同屏幕的大小来使用不同的类名以此来进行区分：
比如：
  • 对于手机端：主体内容不管什么元素，最好都进行铺满，即纵向排列各个元素
  • 对于平板端：主体内容则可以进行一定程度的布局，如菜鸟的二列布局
  • 对于客户端：主体内容则需要进行完整的布局展示，如菜鸟的三列布局
正是因为在三种不同的场景下，需要用到对应的布局效果，因此我们可以通过给同一个元素定义两到三种类名
  1. 手机端：由于所有尺寸均需铺满屏幕，因此我们可以通过属性值选择器来实现，也可以通过类选择器来实现
    • 属性与值选择器：[class*="col-"] 选择所有以 "col-" 为前缀的类，并进行尺寸设置
    • 类选择器：.col-p-* 通过对不同的元素定义不同的类名，再对每一个类名进行尺寸的设置
  2. 平板端：在平板端由于屏幕尺寸要大于手机端，因此可以实现二列布局，也就是所当前的所有元素需要有要给特定的布局尺寸，这里我们可以定义一个新的类名
  3. 客户端：客户端的屏幕尺寸还要更大，因此所有的元素还需要再设置一个新的布局尺寸，这里我们可以定义一个新的类名
按照这种思路，那对于一个 div 标签，我们就可以将其设置两到三种类名：
两种类名
手机端：通过属性与值选择器进行选择
平板端：class="col-m-*" 即column mobile ……
客户端：class-"col-"即column ……
三种类名
手机端：class-"col-pho-*
平板端：class="col-pad-*
客户端：class-"col-*"
